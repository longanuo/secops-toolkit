# cybersecurity-ops 技能使用说明

## 一、技能概述

cybersecurity-ops 是一个综合网络安全运维与竞赛技能助手，覆盖以下七大核心场景：

| 模式 | 用途 | 触发关键词 |
|------|------|-----------|
| 竞赛模式 | CTF / 职业技能大赛解题 | CTF、技能大赛、flag、writeup、Web/Pwn/Reverse/Crypto/Forensics |
| 服务器运维 | 个人Linux服务器安全加固 | 服务器、运维、巡检、日志分析、SSH、应急响应 |
| 沙盒测试 | Docker/VM隔离环境搭建 | 沙盒、Docker、VM、稳定性测试、漏洞复现 |
| 情报学习 | 从GitHub攻防项目学习防御 | GitHub、攻防项目、威胁情报、漏洞库 |
| 防火墙 | nftables从零搭建与优化 | 防火墙、iptables、nftables、规则、DDoS |
| 漏洞验证 | 自动化漏洞扫描与验证 | 渗透、扫描、漏洞、XSS、SQLi、SSRF、XXE、RCE |
| 安全体检 | 系统安全评估与加固 | 体检、评估、评分、加固、审计 |


## 二、使用方法

### 2.1 基本调用方式

直接在对话中描述你的任务即可自动触发对应模式：

示例：
  - "帮我分析这道CTF的Web题"           → 竞赛模式
  - "检查一下我的服务器有没有异常"       → 服务器运维模式
  - "用Docker搭建一个DVWA靶场"         → 沙盒测试模式
  - "分析一下 vulhub 仓库里的CVE复现"   → 情报学习模式
  - "帮我配置nftables防火墙"           → 防火墙模式
  - "扫描这个URL有没有漏洞"             → 漏洞验证模式
  - "做一个完整的安全审计"               → 安全体检模式

### 2.2 多模式组合

某些任务会同时触发多个模式：
  - "从GitHub学习攻防然后更新防火墙规则"  → 情报学习 + 防火墙
  - "在沙盒里复现CVE并写writeup"         → 沙盒测试 + 竞赛模式
  - "扫描漏洞并生成防御规则"             → 漏洞验证 + 防火墙


## 三、各模式详细说明

### 3.1 竞赛模式 (competition.md)

覆盖题型：Web安全、Pwn二进制利用、Reverse逆向工程、Crypto密码学、
         Forensics取证分析、Misc杂项

快速题目识别：
  给了 URL / 网页         → Web 安全
  给了二进制 + nc地址     → Pwn
  给了可执行文件无交互    → Reverse
  给了密文/数学公式       → Crypto
  给了内存镜像/流量包     → Forensics
  给了奇怪文件/脑洞题     → Misc

常用工具栈：
  Web:       Burp Suite, SQLMap, Dirsearch, Gobuster, XSStrike
  Pwn:       pwntools, GDB+peda/pwndbg, ROPgadget, checksec
  Reverse:   Ghidra, IDA Free, x64dbg, strings, file, binwalk
  Crypto:    CyberChef, SageMath, hashcat, John the Ripper, RsaCtfTool
  Forensics: Autopsy, Volatility, Wireshark, steghide, foremost, exiftool
  Misc:      CyberChef, Python3, pycryptodome, z3-solver

内置技术详解：
  - SQL注入绕WAF技巧
  - XSS payload构造
  - 文件包含 / 路径穿越
  - SSTI模板注入
  - 文件上传绕过
  - 栈溢出 / ret2libc / ROP链
  - 格式化字符串漏洞
  - 堆利用（unsorted bin attack）
  - 常见加密攻击（RSA/ECDSA/哈希）
  - 内存取证 / 流量分析 / 隐写术


### 3.2 服务器运维模式 (server-ops.md)

覆盖场景：
  - 新服务器初始化安全加固（4步流程）
  - 账户认证加固（SSH密钥、禁root登录）
  - 系统更新与最小化服务
  - 审计与监控（auditd、fail2ban）
  - 内核加固（sysctl参数）
  - 日常巡检（健康检查脚本）
  - 日志分析（SSH暴力破解统计）
  - 应急响应（入侵排查流程）

关键命令：
  lynis audit system              # 系统安全审计
  fail2ban-client status sshd     # 查看封禁状态
  ss -tlnp                        # 查看监听端口
  last | head -20                 # 最近登录记录


### 3.3 沙盒测试模式 (sandbox-testing.md)

选型建议：
  需要完整OS内核隔离         → KVM/QEMU
  需要轻量快速启动           → Docker
  需要系统容器比Docker更真实  → LXC

内置模板：
  - Docker安全沙盒配置（资源限制 + 网络隔离）
  - CTF靶机环境 Docker Compose（攻击机 + 靶机）
  - 一键搭建 DVWA / Vulhub / Pikachu 环境
  - 服务器稳定性压测（stress-ng / fio / wrk）


### 3.4 情报学习模式 (github-intel.md)

核心理念：从攻击技术中提取防御视角

推荐仓库：
  漏洞复现:   vulhub/vulhub
  Payload:    PayloadsAllTheThings, SecLists
  检测模板:   nuclei-templates
  系统加固:   lynis, ansible-collection-hardening
  威胁情报:   firehol/blocklist-ipsets, stamparm/ipsum
  CTF资源:    cf-wiki, NSSCTF

自动化工具：
  nuclei -target http://your-server -severity high,critical
  # 直接使用 nuclei 模板库进行漏洞扫描


### 3.5 防火墙模式 (firewall-setup.md)

推荐方案：nftables（Linux 5.2+ 内核原生）

特色功能：
  - 多源威胁情报自动更新（stamparm/ipsum, firehol, maltrail）
  - IP黑名单集合（自动合并去重）
  - 暴力破解临时封禁（SSH 3次失败封禁，1小时自动解封）
  - 速率限制防DDoS
  - 管理员IP白名单
  - 端口保护策略
  - CrowdSec社区威胁情报集成

管理命令：
  secops --fw-status      # 查看防火墙状态
  secops --fw-audit       # 审计防火墙规则
  secops --update-firewall # 更新威胁情报


### 3.6 漏洞验证模式 (vuln-scan.md)

支持的漏洞类型：
  XSS      - 反射型/存储型跨站脚本
  SQLi     - 报错型/布尔盲注/时间盲注
  SSTI     - 模板注入 (Jinja2/Twig/FreeMarker)
  LFI      - 本地文件包含/路径穿越
  SSRF     - 服务端请求伪造
  XXE      - 外部实体注入
  RCE      - 命令注入
  NoSQLi   - MongoDB/Redis 注入
  InfoLeak - 敏感路径/调试接口泄露

使用方法：
  secops --scan https://target.com    # Nuclei 扫描
  # 或使用 Python API
  from secops.attack_engine import AttackEngine
  engine = AttackEngine("https://target.com")
  engine.run_all(modules=["xss", "sqli", "ssrf"])


### 3.7 安全体检模式 (health-check.md)

完整审计内容：
  1. 系统安全体检 (SSH配置、服务状态、文件权限)
  2. 防火墙规则审计 (端口保护、管理员白名单)
  3. 威胁情报状态 (恶意IP数量、更新时间)

使用方法：
  secops --full-audit      # 完整审计
  secops --check           # 仅系统体检
  secops --intel-status    # 情报状态


## 四、授权边界（强制遵守）

所有渗透测试、漏洞验证、攻击技术演练仅在以下环境执行：
  ✓ 自己拥有的沙盒/测试服务器
  ✓ 明确授权的靶机平台（HackTheBox、TryHackMe、BUUCTF、NSSCTF等）
  ✓ 本地Docker/VM隔离环境

绝不对未授权目标执行任何探测或攻击！


## 五、防御优先思维

学习攻击技术的目的是构建更好的防御。每次学习攻击手法后思考：
  1. 这个漏洞的检测特征是什么？
  2. 如何在防火墙/WAF层拦截？
  3. 如何在应用层修复？
  4. 如何生成对应的监控告警规则？

## 七、典型使用场景示例

场景1：分析CTF题目
  输入："这道题给了一个URL http://xxx，帮我看看"
  输出：目录扫描 → 漏洞检测 → 利用步骤 → writeup + JSON日志

场景2：服务器安全加固
  输入："新买的Ubuntu服务器，帮我做安全加固"
  输出：SSH加固 → 防火墙配置 → fail2ban → auditd → 巡检脚本

场景3：搭建CTF靶场练习
  输入："帮我搭建一个Web漏洞练习环境"
  输出：Docker Compose配置 → 一键启动 → 访问指南

场景4：从GitHub学习并更新防火墙
  输入："分析vulhub里的最新CVE，然后更新我的防火墙规则"
  输出：CVE分析 → 检测特征提取 → nftables规则更新

场景5：完整安全审计
  输入："做一个完整的安全审计"
  输出：系统体检 + 防火墙审计 + 威胁情报状态 → 综合评分


## 八、注意事项

1. PDF读取：首次使用需安装 pdfplumber（pip install pdfplumber）
2. Windows路径：Python中使用 r"D:\..." 原始字符串
3. 中文字体：matplotlib 设置 SimHei / Microsoft YaHei
4. 文件输出：所有产出物必须写入磁盘，不仅在对话中显示
5. 多解法验证：预测/估计类问题至少用2-3种方法加权平均
