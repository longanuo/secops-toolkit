# 模块架构总览

## 外部指挥大模型调度机制

Hermes Agent 作为外部指挥大模型，通过以下流程调度本 skill：

1. 用户输入任务 → Hermes Agent 识别触发关键词
2. 路由到对应模式 → 加载 references/ 下的参考文档
3. 执行任务 → 输出结构化 JSON（含 habit_hints）
4. 习惯学习引擎 → 更新 user_habits.json（工具偏好、工作流模式、纠错记录）
5. 下次调用 → 将用户偏好注入 system prompt，实现个性化

调度链：[用户请求] → [Hermes Agent] → [模式识别路由表] → [cybersecurity-ops skill] → [结构化JSON输出] → [Hermes 记忆更新] → [偏好注入]

## Skill 1: competition（竞赛模式）

### 目标与描述
覆盖 CTF / 职业技能大赛网络安全赛道全方向解题，包括 Web、Pwn、Reverse、Crypto、Forensics、Misc 六大题型。提供从题目识别（30秒分类）到完整解题的全流程指导。

### 原始代码/指令
**触发关键词：** CTF、技能大赛、题目、flag、writeup、Web/Pwn/Reverse/Crypto/Forensics

**核心能力：**
- Web安全：目录扫描（dirsearch/gobuster）、SQL注入（sqlmap+WAF绕过）、XSS payload构造、文件包含/路径穿越、SSTI模板注入、文件上传绕过
- Pwn：checksec保护机制识别、pwntools模板（栈溢出/ret2libc/ROP链）、格式化字符串漏洞利用
- Reverse：Ghidra/IDA静态分析、加壳检测与脱壳、常见算法识别（异或/Base64变表/RC4）、脚本自动还原
- Crypto：RSA快速工具箱（RsaCtfTool）、常见场景速查（小明文攻击/共模攻击/Fermat分解/Wiener攻击）、古典密码识别
- Forensics：文件分析（binwalk/foremost）、内存取证（Volatility）、流量分析（Wireshark）、隐写术
- Misc：编码识别、数据提取、脑洞题解法

**工具栈：** Burp Suite, SQLMap, pwntools, GDB+pwndbg, Ghidra, IDA, CyberChef, Volatility, Wireshark 等

## Skill 2: server-ops（服务器运维模式）

### 目标与描述
个人 Linux 服务器安全加固与日常运维，覆盖新机初始化（4步流程）、账户认证加固、审计监控、内核加固、日常巡检和应急响应。

### 原始代码/指令
**触发关键词：** 服务器、运维、巡检、日志分析、加固、SSH、应急响应

**核心能力：**
- 初始化加固：SSH密钥认证+禁root登录、自动安全更新、auditd审计+fail2ban、sysctl内核加固（防IP欺骗/ICMP重定向/SYN Flood）
- 日常巡检：health_check.sh 脚本（CPU/内存/磁盘/异常进程/登录记录/监听端口/定时任务）
- 日志分析：SSH暴力破解统计、sudo使用记录、Nginx扫描特征识别
- 应急响应：入侵取证（不关机先快照）、后门检查（隐藏进程/SUID文件/新增账户）、Webshell检测
- 性能监控：基线记录脚本、Netdata实时监控

## Skill 3: sandbox-testing（沙盒测试模式）

### 目标与描述
Docker/VM/LXC 隔离环境搭建与服务器稳定性测试，用于漏洞复现、CTF靶场练习和压测。

### 原始代码/指令
**触发关键词：** 沙盒、Docker、VM、稳定性测试、漏洞复现、隔离环境

**核心能力：**
- 沙盒选型：KVM/QEMU（完整OS隔离）→ Docker（轻量快速）→ LXC（系统容器）
- Docker安全沙盒：资源限制（memory/cpu/pids）+ 网络隔离（sandbox-net）+ 安全选项（no-new-privileges/read-only/cap-drop）
- CTF靶场：Docker Compose 一键搭建（target-web + attacker 双容器，internal网络隔离）
- 一键搭建：DVWA、Vulhub、Pikachu、WebGoat
- 稳定性压测：stress-ng（CPU/内存）、fio（磁盘I/O）、wrk/ab/siege（Web并发）、iperf3（网络）
- 漏洞复现流程：隔离网络启动 → 利用验证 → 记录检测特征 → 提取防御规则 → 清除环境

## Skill 4: github-intel（情报学习模式）

### 目标与描述
从 GitHub 攻防项目自动学习攻击技术，提取防御视角，生成防火墙/WAF/监控规则。核心理念："学习攻击的目的是构建更好的防御"。

### 原始代码/指令
**触发关键词：** GitHub、攻防项目、学习、自动爬取、威胁情报、漏洞库

**核心能力：**
- 推荐仓库：vulhub/vulhub（漏洞复现）、PayloadsAllTheThings（Payload集）、nuclei-templates（检测模板）、lynis（系统加固）、firehol/blocklist-ipsets（威胁情报IP）
- 自动化扫描：nuclei -target + severity 过滤
- 攻击→防御转化框架：XSS→WAF string match、SQL注入→WAF规则、路径穿越→Nginx规则、扫描行为→fail2ban、恶意IP→nftables set
- 情报数据库维护：恶意IP自动更新（stamparm/ipsum）、30天滚动保留、7天合并去重
- Cron定时学习：每天凌晨更新威胁情报并应用到防火墙，每周更新nuclei模板扫描自己的服务

## Skill 5: firewall（防火墙模式）

### 目标与描述
nftables 防火墙从零搭建与规则智能优化，含 IP 黑名单自动更新、暴力破解封禁、DDoS 速率限制、CrowdSec 社区情报集成。

### 原始代码/指令
**触发关键词：** 防火墙、iptables、nftables、规则、封堵、DDoS

**核心能力：**
- 推荐方案：nftables（Linux 5.2+ 内核原生，替代 iptables）
- IP黑名单集合：自动从多个威胁情报源更新（stamparm/ipsum、firehol/blocklist-ipsets）
- 暴力破解防护：SSH 3次失败封禁、1小时自动解封（nftables set + meter）
- 速率限制：防 DDoS 连接速率限制
- CrowdSec集成：社区驱动的威胁情报共享
- 自动更新脚本：/opt/cybersec/update_firewall.sh

## Skill 6: hermes-integration（Agent 集成模式）

### 目标与描述
定义 cybersecurity-ops 与 Hermes Agent 的集成规范，包括统一输出 Schema、Habit Learning 数据结构、工具调用接口设计、偏好注入机制和错误处理规范。

### 原始代码/指令
**触发关键词：** Hermes、agent、JSON输出、工具调用、习惯学习、自动化流程

**核心能力：**
- 统一输出 Schema（$schema: cybersecurity-ops/v1）：mode、task_id、execution（steps/commands/duration）、artifacts、findings（severity分级）、habit_hints、next_actions、tags
- Habit Learning 数据结构（user_habits.json）：用户画像（skill_level/primary_focus）、工具偏好（按模式+子类型）、工作流模式（典型步骤/平均耗时/成功率）、防火墙配置、学习历史、纠错记录
- 工具调用接口：solve_ctf_challenge()、server_health_check()、sandbox_test()、learn_from_github()、update_firewall()
- 偏好注入：build_skill_prompt() 将用户习惯注入下次调用的 system prompt
- 记忆更新钩子：update_habits() 根据任务输出更新工具偏好、工作流模式、纠错记录、防火墙配置
- 错误处理：AUTH_VIOLATION（超授权范围）、SANDBOX_ESCAPE（容器逃逸）、TOOL_NOT_FOUND、NETWORK_ERROR

## 全局约束

1. **授权边界（强制）**：所有渗透测试仅在自有沙盒/授权靶机（HTB/TryHackMe/BUUCTF/NSSCTF）/本地隔离环境执行
2. **防御优先思维**：每次学习攻击后必须回答——检测特征？WAF拦截？应用层修复？监控告警规则？
3. **可重现性**：所有操作可记录、可重现、可自动化
4. **多模式组合**：支持同时触发多模式（如"从GitHub学习→更新防火墙" = 情报学习+防火墙）
